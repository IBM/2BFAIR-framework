  • Access Conditions: Requirements that a machine can understand to either automatically execute the requisite steps before accessing the Data, or alert the user the existence of such requisites (for instance, the need to create an account).
  • Assessment: The process of estimating how a Digital Object takes the FAIR Principles into consideration. Can be either a Characterization or an Evaluation.
  • Atomic Digital Object: A Digital Object that cannot be further subdivided as other Digital Objects.
  • Characterization: The process of answering a questionnaire and ending up with descriptions of the ways in which a Digital Object complies with the FAIR Principles.
  • Community Standard: A set of specifications or styles for Data or Metadata that are widely accepted within the particular community doing the FAIRness Assessment.
  • Composite Digital Object: A Digital Object that can be subdivided into a set of either Atomic or further Composite Objects. A primary example of Composite Digital Objects are Datasets.
  • Data: See Digital Object.
  • Dataset: A set of Data. Thus, for FAIRness Assessment purposes, a set of Digital Objects. Datasets are the primary examples of Composite Digital Objects.
  • Digital Object: A sequence (or a set of sequences) of bits, incorporating information (such as observations or measurements) pertaining some stakeholder. Each of the sequences should ideally be structured in a way that is machine-interpretable and have as an essential element an associated GUPRI. Digital Objects are Resources, and can be either Atomic or Composite.
  • Evaluation: The process of running a series of Tests on a Digital Object and giving it a FAIRness grade.
  • FAIR Implementation Options: An optional set of parameters derived from a scientific community’s choices regarding FAIRness Assessment that is used to calibrate a FAIRness Evaluation tool. It is created by answering a questionnaire, ideally by one or more specific communities, for a particular task, and collectively during a workshop by experts with different Respondent Profiles.
  • FAIR Principles: A set of 15 Principles with which scientific Data and Metadata should comply. They are not directly equated with the quality of Data and the quality of Metadata, but can help improve them. They can be divided into four dimensions, each comprised of a subset of Principles designated by one letter of the acronym (Findability, Accessibility, Interoperability, Reusability). Principles are elaborated as Indicators.
  • FAIRness: A percentage grade indicating how close a digital object is to fully abiding by the FAIR Principles. Each of its 4 constitutive scores (for the F, A, I, and R dimensions) can be individually explored.
  • Globally Unique Identifier: An Identifier that is guaranteed to uniquely identify a particular Resource, irrespective of the context. It is thus impossible for such an Identifier to refer to different resources. For optimal FAIRness, this should be a GUPRI. Not to be confused with the similarly named internet standard UUID/RFC4122.
  • Globally Unique Persistent Resolvable Identifier (GUPRI): A specialized Identifier that is Globally Unique, Persistent and Resolvable, considered optimal for FAIR compliance.
  • Identifier: An attribute that uniquely identifies a Resource in a given context. For optimal FAIRness, this should be a GUPRI.
  • Indicator: A specific and unambiguous way of describing one aspect that needs to be considered for FAIRness Assessment. Indicators are domain-agnostic. Indicators have priorities (essential, important or optional). Indicators are implemented as Metrics.
  • License: A legal document that specifies what a user can do with a Resource, probably including ways in which it can be Reused.
  • Metadata: Metadata has a purely nominalist definition, which means Metadata is anything that is explicitly characterized as Metadata in the system. Metadata can be an independent Digital Object (which is preferred according to the FAIR Principles) or it can be part of the Digital Object under Assessment. One can be interested in looking at a specific Metadata Record as a Digital Object, but in this case FAIR Assessment begins anew and the fresh Digital Object under evaluation will have its own specific Metadata.
  • Metadata Element (or Metadata Attribute): A single property that is used to label a characteristic (for example, Dublin Core’s dc:title, Schema.org’s schema:name or DataCite’s Title).
  • Metadata Record: A set of Metadata Statements identified as a group. Ideally, one Record pertains to one Digital Object.
  • Metadata Statement: A single characteristic of a Resource, usually expressed as a property-value pair (for example, Title=“My data”).
  • Metric: A way of describing what will be actually measured. Metrics can be domain-specific, and include the alternative of being irrelevant for specific domains. Metrics have numeric values and are related to the Indicators that they implement. Each Metric is evaluated using one or more Tests.
  • Ontology: A formalised set of concepts relevant to a particular area of interest, representing rich and complex knowledge about things, groups of things, and relations between things, as well as a set of constraints about the usage of its terms.
  • Persistent Identifier (PID): An Identifier that has received an institutional commitment guaranteeing it to remain valid over a period of time. During this time, the publisher or maintainer of the Identifier promises that it will continue to Resolve to the same Resource. For optimal FAIRness, this should be a GUPRI.
  • Principle: A set of guidelines that are deliberately general, broad, loose, ambiguous enough to lead to different interpretations. Principles are elaborated as Indicators.
  • Resolvable Identifier: An Identifier that can undergo a process in which it is the input — a request — to a network service to receive in return a specific output of one or more pieces of current information (state Data, that is, a Digital Object) related to the identified entity (for example, a URL). For optimal FAIRness, this should be a GUPRI.
  • Resource: Anything that is accessed and/or reused and of which the FAIRness is being assessed, including Metadata Records and Datasets.
  • Respondent Profile: A category to describe the type of expert that was involved in creating a set of FAIR Implementation Options by responding a questionnaire. The skills represented are: Subject Matter Expertise, Software Engineering, Data Modeling, Database Administration.
  • Reuse: The act of using an existing Resource for a different purpose or in a different context. This may involve republishing and creating derivatives as far as allowed under the License specified for reuse of the Resource.
  • Taxonomy: A schema for the orderly classification of things according to their relationships. Taxonomies are usually hierarchical.
  • Test: A practical solution to evaluate each Metric. Tests are usually of the type pass/fail. Test results contribute not only to FAIRness Evaluation but also indicate ways of improving the former.
  • Thesaurus: A controlled vocabulary that seeks to dictate semantic manifestations of Metadata in the indexing of Digital Objects. Usually takes the form of a list of subject headings or descriptors with a crossreference system, sometimes as a hierarchy of broader and narrower terms, sometimes simply as lists of synonyms, quasi-synonyms and antonyms.
  • Vocabulary: A set of terms or concepts that can be used in the description of a Resource. This includes Thesauri, Taxonomies, and Ontologies.

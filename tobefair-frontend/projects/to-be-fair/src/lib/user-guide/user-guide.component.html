User Guide

The 2BFAIR-frontend is divided into eight pages:
  • Home: Start page where the user supplies the digital object identifier to start its FAIRness evaluation.
  • Result: Visual representation of the overall FAIRness result.
  • Explorer: Overview of the evaluation for each dimension. We define F, A, I, and R as FAIR dimensions.
  • Details: Details of the evaluation for principles, metrics, and tests.
  • FAIR Glossary: Presents the definition of the FAIR concepts used in the tool development.
  • Tool Glossary: Presents the explanation of some concepts used in 2BFAIR.
  • User Guide: Explanations about this page and the functionality available in the 2BFAIR-frontend.
  • Full Report: Combination of the content from the Result, Explorer and Details pages.

  • Home

    1. The Home page is composed of four elements:
      1. A text box to enter the digital object's identifier, which is usually a URI, URL, or GUID.
      2. A button to run the evaluation.
      3. A link to run an example evaluation.
      4. An additional resources section.
    2. To start an evaluation fill in the text box with the digital object identifier and click the button Start an evaluation.


  • Result

    1. The page Result is composed of three elements:
      1. A badge that is a graphic representation of the overall FAIRness assessment.
      2. The URI that was entered as the digital object's identifier.
      3. A button to go to the Explorer page
    2. Each badge's column represents the FAIRness evaluation result achieved by the digital object corresponding to a FAIR dimension. The number on the right of the badge represents the total score as a percentage achieved by the digital object, i.e., it considers all evaluated FAIR dimensions.

    3. To learn more about your result, click the button FAIRness Explorer to navigate to the page Explorer.

  • Explorer

    1. The page Explorer is composed of 4 elements:
      1. Header: Presents information about the evaluated object, a link to download the evaluation result, and the FAIRness badge.
      2. Graph: Represents the score as a percentage achieved for each dimension with boxes representing the result achieved for each evaluated metric within the dimension.
      3. Legend: Explains the importance of each metric
      4. Priority Recommendations: Lists the most important recommendations to improve the FAIRness of the digital object.
      2. These elements are detailed as follows.
    2. The header is composed of 3 elements:
      1. The identifier of the evaluated resource.
      2. A link to download the FAIRness result in JSON format.
      3. The badge with the FAIRness result.
    3. The graph presents, for each dimension, the metrics used to evaluate the digital object presented as a rectangle and the score as a percentage achieved by the dimension evaluation. Next to each dimension, there is an arrow that, if clicked, opens the Details page in the section corresponding to the dimension. Each rectangle depicts:
      1. The metric ID in the middle of the rectangle.
      2. The FAIRness result of the metric, which ranges from 0 (the rectangle is not filled) to 1 (the rectangle is fully filled).
      3. The importance of the metric represented by three color tones.
      4. Even though the metric's score is 0, the border is presented to depict the metric importance.
    4. The legend represents the importance of the metric.
    5. Priority recommendations shows the most important recommendations to follow to improve the FAIRness level of the digital object. If there are more than n priorities, where n is defined in the 2BFAIR configuration, a See more link is presented. When the user click son this link, it opens the Details page in the dimension corresponding to the first recommendation.

  • Details

    1. The page Details is composed of two elements:
      1. The same header as in the Explorer page.
      2. An accordion with the evaluation details, which fields are explained as follows.
    2. The accordion is divided into three instances:
      1. Principle
      2. Metrics
      3. Test
    3. For each principle, the following are presented:
      1. Name: the definition of the principle.
      2. Score: a value between 0 and 1 representing the average of the scores of all the principle's metrics score.
    4. For each metric, the following are presented:
      1. Name: the definition of the metric following FAIRsFAIR metric definition.
      2. Maturity
      3. Score: A value between 0 and 1 representing the average score achieved by all the FAIRness tests of the metric.
      4. Score Mechanism: Defined here, in the tool glossary.
    5. For each test, the following are presented:
      1. Status: It can be passed, failed or not executed. The status is passed when the digital object passes in the test evaluation, and failed if not. The status is not executed when the test doesn't run because of missing requirements.
      2. Score: A value between 0 and the maximum test score, which varies from test to test.
      3. Results: A summary of the test's result.
      4. Results details: An optional field that provides details about the test execution.
      5. Recommendations: Suggestions about what should be done to achieved a higher FAIRness score. Presented even if the test passes.
      6. Recommendations details: An optional field that provides details about the recommendation, e.g., steps on how to implement the improvement.
      7. Losses: What is lost if the digital object doesn't pass the test.

  • FAIR Glossary

    1. The page FAIR Glossary is composed of:
      1. Sidebar: Allows to browse commonly used terms in the FAIR concepts.
      2. Definitions:Definitions of each term
    2. Links are included in all terms present in the glossary that appear on 2BFAIR's web pages so that the user can access the FAIR Glossary by clicking on the word. Self-references are also included to navigate through cross-terms in the glossary.

  • Tool Glossary

    1. The page Tool Glossary is composed of:
      1. Sidebar: Allows to browse commonly used terms in the FAIR concepts.
      2. Definitions:Definitions of each term
    2. Links are included in all terms present in the glossary that appear on 2BFAIR's web pages so that the user can access the FAIR Glossary by clicking on the word. Self-references are also included to navigate through cross-terms in the glossary.

  • Full Report

    1. The Full Report page is a compilation of the Result, Explorer and Details pages and is composed of five elements:
      1. Download ButtonAllows the user to download the content of the page.
      2. Header: Presents information about the evaluated object and a FAIRness badge. (Click here for more details)
      3. Overall Results: Shows the FAIRness badge of your data and the URI entered as the digital object's identifier. (Click here for more details)
      4. Result by Metric Shows the results for each dimension divided by metrics. (Click here for more details)
      5. Lists priority recommendations for improvements
      6. Detailed Results Provides information about each test, metric and principle. (Click here for more details)
    2. The download button allows the user to download a PDF file with the evaluation result.
